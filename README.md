Proxy Service CitadelProxy Service — это комплексная экосистема для управления и использования прокси-доступа. Включает в себя веб-панель управления, асинхронный бэкенд и десктопный клиент с обновлением статуса в реальном времени.
🚀 Основные возможности
Автоматизация: Регистрация и мгновенная отправка ключей через Celery + Mailtrap.
Гибкое управление: Личный кабинет для обновления ключей и смены пароля.
Real-time: Отслеживание статуса подключения через WebSockets в десктопном приложении.
Умное распределение: Автоматический поиск свободных "виртуалок" для пользователей.
Полная изоляция: Весь стек разворачивается одной командой через Docker.

🛠 Технологический стек
Backend - FastAPI, SQLAlchemy 2.0, PostgreSQL 15
Frontend (Web)Vue 3, Vuetify 3, Pinia
Desktop App PyQt5, WebSocketsА
синхронность Redis 7 + Celery 


1. git clone https://github.com/your-repo/proxy-service.git
cd proxy-service
cp env.example backend/.env
2. Запуск инфраструктуры docker-compose up -d --build

3. Автоматическая настройка базы данных
При первом запуске docker-compose автоматически применяются миграции Alembic и наполняется база данных.

Система по умолчанию создает набор тестовых виртуальных машин (Proxy Europe, USA, Asia) через скрипт init_db.py, который запускается внутри контейнера бэкенда.

🖥 Использование десктоп-клиента
Установите зависимости: pip install -r requirements.txt.
Перейдите в папку клиента: cd desktop-app.
Запустите приложение: python main.py.
Авторизация: Вставьте ваш JWT токен (можно взять из LocalStorage браузера) и Ключ активации из письма.


Полная документация доступна после запуска по адресу: http://localhost:8000/docs 
Тестирование
4. Для запуска тестов внутри контейнера используйте:

docker exec -it proxy_backend python -m pytest tests/ -v
